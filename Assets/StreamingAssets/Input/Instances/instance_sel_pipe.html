Generate instance order for task input (i.e. param2.txt file for game)

Instance order is generated in such a way that each block has the same number of instances of each type. Therefore, nInstances must be a multiple of binFactor.

### TSP Instance order generation (param2.txt)
import sys
#print (sys.version)
work_dir=r.work_dir
sys.path.insert(0, work_dir)
seed=r.seed
import instanceSelctionFunctions as isf
folderOut_sub=r.folderOut_sub
#folderOut="/Users/jfranco1/Google Drive/Melbourne/UNIMELB/Complexity Project/Code/Instance Selection/ICX3/3-SAT/Instance Selection/Output/subsample/"#r.folderOutput#
nOrderRandomizationsMin=1#0#1
nOrderRandomizations=50
tN=8
bN=3
nTypes=6
binFactor=8 # = sum(sampleSizePerBin) when sizePerBin=1
nInstances=tN*bN
sizePerBin=int(nInstances/binFactor)
#Total number (Including all blocks) instances per Type
sampleSizePerBin=[sizePerBin,sizePerBin,sizePerBin,sizePerBin,sizePerBin*2,sizePerBin*2] 
# Samples randomly from each instance-type sampleSizePerBin
# Assumes that the instance number is generated in order of increasing type; e.g. instance type 1-6
# Output: list of sublists. Each sublist has sampleSizePerBin size with the instances ID
for i in range(nOrderRandomizationsMin,nOrderRandomizations+1):
    instanceOrder=isf.generateInstanceOrder(tN, bN,sampleSizePerBin,seed+i)
    isf.exportTaskInfo(tN,bN,instanceOrder,nInstances,folderOut_sub,i) #Exports 'param2.txt' with the required input for the task
print(str(i) + " Instance Order generation parameters were generated.")
